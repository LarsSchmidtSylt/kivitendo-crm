XML/RPC-Service für LX-Office CRM

Momentaner Stand

Der XML/RPC-Service unterstützt momentan den lesenden/schreibenden Zugriff auf die Kontaktdaten von LX-Office (Benutzerdaten nur lesend). Die Schnittstelle wurde so geschaffen, dass der Zugriff via Kaddressbook möglich ist. KDE seitig wurde diese Schnittstelle ursprünglich für den Zugriff auf die Groupware eGroupware implementiert. Nähere Informationen findet man auf den jeweiligen Internetseiten der beiden Projekten.

Die getestete Kaddressbook Version ist 3.5.3. Prinzipiell sollte es aber auch mit „älteren“ Versionen funktionieren. Die LX-Office Version ist 2.4.2. Auch hier sollte es abwärtskompatibel sein, solange sich die Datenbankstruktur nicht geändert hat.

Installation und Konfiguration

  • Das tgz-Archiv entpacken (am Besten in das CRM-Verzeichnis).

  • Benutzerrechte ändern: Bsp. chown -R wwwrun:www services, chmod -R 644 services

  • In der Datei header.inc.php den Pfad zur LX-Office ERP Root anpassen.

  • Anschließend die Verbindungseinstellungen in KAddressbook vornehmen. Ein hilfreicher Artikel findet man hier http://www.pro-linux.de/berichte/kontact_egroupware.html. Es müssen hierbei nur die URL (mit http:// am Anfang !!!), der ERP-Username, sowie das ERP-Passwort angegeben werden.
    Die URL muss dann auf die Datei xmlrpc.php verweisen.

  • Das war es. Jetzt sollte man eigentlich schon auf die Kontaktdaten zugreifen können.

Hinweis: Dieser Service ist noch experimentell. D.h. Benutzung auf eigene Gefahr ;-) !
